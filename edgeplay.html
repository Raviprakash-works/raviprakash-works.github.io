The Problem

Manual MPV Workflows Crushing ML Velocity

Computer vision engineers manually trimmed 50GB+ raw sports footage using desktop MPV players and passed static video clips over physical drives or ad-hoc download links. Annotation tasks were assigned manually without lineage or version control.

This fragmented process caused multi-day turnaround delays, redundant storage expenses, and zero auditability when training computer vision models for sports analytics.

FastAPI Schema & PRD Specification

PRD-001 EdgePlay.ai — Modular API Backend & Automated Pipeline

1. Product Scope: Defined schema and worker execution pipelines for a modular FastAPI backend spanning 9 core entities: Projects, Users, Assets, Tasks, Jobs, Workers, Assignments, Events & Storage.

2. Browser Trimming Workspace: Specified a custom web-based trimming interface replacing desktop MPV software, enabling annotators to set frame-accurate start/end timestamps directly in browser.

3. Embedded Annotation Layer: Planned and specified Label Studio integration as an embedded iframe layer, maintaining EdgePlay.ai as the single system-of-record for metadata and asset lineage.

4. Media Pipeline: Architected automated media processing using signed B2 URLs, FFmpeg-based worker execution, and version-controlled asset registration.
