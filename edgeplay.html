The Problem

Manual Workflows Crushing ML Velocity

Raw match footage (50GB+ per match) was stored on physical drives or passed around via temporary download links. Computer Vision engineers spent 4+ hours per match manually searching, downloading, and trimming footage into clips before annotation could even begin.

Workflows were fragmented across email, local drives, and spreadsheets. Annotations had zero version control, redundant storage costs inflated cloud bills, and dataset reproducibility was near impossible.

Feature Module

Cricket Video Annotation Module

The cricket video annotation workflow is embedded as a core module within EdgePlay. It automatically detects delivery windows (10-second clips around bowler release), slices footage, and generates pre-signed B2 URLs.

The database schema links raw match metadata to individual delivery clips and annotator tasks, ensuring zero data duplication and full traceability from raw video to trained model weight.

PRD Specification

PRD-001 Automated Video Annotation Queue & Cloud Sync

1. Problem Statement: ML engineers manually trimmed 50GB+ match footage files, causing 4-hour delays per match and zero reproducibility across annotation batches.

2. Target User Persona: Data Annotators & CV Engineers requiring 60fps pre-trimmed, version-controlled clips.

3. RICE Score: 8.4 / 10 — Reach: 100% team, Impact: High (50% turnaround reduction), Confidence: 90%, Effort: 4-day sprint.

4. Functional Requirements: Auto-detect frame rates & transcode to 1080p/60fps. Slice into 10-second event clips at delivery timestamps. Generate Backblaze B2 pre-signed URLs with 24hr TTL. Auto-sync annotation tasks with Label Studio REST API.

5. Success Metric: Reduce dataset preparation time from 4 hours to under 10 minutes per match.

Trade-Off Analysis

Product Decisions & Architecture Trade-Offs

Decision 01

Why Backblaze B2 over AWS S3?

75% lower egress costs with S3-compatible API. AWS S3 offered unnecessary enterprise features for single-project scale while incurring significantly higher data transfer costs.

Decision 02

Why 10-Second Event Clips?

Optimal annotation window for sports events. Balances enough visual context (bowler run-up + delivery + follow-through) with annotation speed and precise frame boundary labeling.

Decision 03

Why Label Studio Integration?

Open-source platform with robust REST API for task automation. Natively supports YOLO export format, eliminating custom converter scripts and vendor lock-in.
