01 / Problem Statement

The Bottleneck in Sports AI Development

Building computer vision models for sports like cricket involves an enormous operational burden. While capturing raw match footage is easy, preparing high-quality, frame-accurate training datasets is extremely difficult.

Before EdgePlay, raw footage (50GB+ per match) lived across physical hard drives. ML engineers spent up to 4 hours manually searching for specific events, trimming clips, and sending files to annotators via ad-hoc tools. This fragmented workflow caused severe storage waste, version control failures, and delayed model training cycles.

03 / Product Strategy

Cloud-First System Architecture

Instead of building another isolated annotation utility, EdgePlay was envisioned as an integrated operating system for sports AI data. The raw footage enters the cloud, is automatically segmented, assigned to queue workers, annotated, and exported into training datasets.

Cloud Pipeline Workflow:
[ Raw Match Video (50GB) ] ➔ [ Cloud Ingest Trigger ] ➔ [ Automated FFmpeg Event Trimming (10s Clips) ] ➔ [ Backblaze B2 S3 Pre-Signed URLs ] ➔ [ Label Studio Queue Auto-Population ] ➔ [ Quality Assurance Review ] ➔ [ COCO/YOLO Dataset Export ] ➔ [ Model Training Pipeline ]
04 / Feature Module

Cricket Video Annotation & Telemetry Module

As part of EdgePlay, a specialized cricket annotation module was built to handle sport-specific telemetry (bowling release points, pitch bounce locations, shot direction vectors, and boundary events).

Key Product Capabilities:

  • Automated Event Trimming: Slices full 3-hour match recordings into 10-second delivery clips based on audio spikes and umpire signaling telemetry.
  • Pre-Signed B2 Streaming: Streams trimmed clips directly into browser-based annotation interfaces without downloading gigabytes locally.
  • Immutable Dataset Versioning: Every annotated clip is assigned a unique SHA-256 hash and metadata tag, ensuring 100% training reproducibility.
05 / PRD & Specification

Product Requirements Document (PRD) Snippet

PRD-001 EdgePlay AI — Automated Video Annotation Queue & Cloud Sync

1. Problem Statement: ML engineers manually trimmed 50GB+ raw match files, creating 4-hour operational delays before annotators could start labeling.

2. User Persona: Data Annotators & Computer Vision Engineers requiring standardized frame rates (60fps) and pre-trimmed clips.

3. Prioritization (RICE Score = 8.4/10): High Reach (100% of video ingestion), High Impact (50% turnaround reduction), High Confidence (90%), Low Effort (4-day sprint using FFmpeg + B2 API).

4. Functional Requirements:
• Auto-detect video frame rates and transcode to 1080p 60fps upon cloud upload.
• Slice raw match videos into 10-second event clips around detected delivery timestamps.
• Generate Backblaze B2 S3 pre-signed URLs valid for 24-hour annotation sessions.
• Auto-sync task payload with Label Studio REST API.

5. Non-Functional Requirements: System must handle 500+ simultaneous clip URL generation requests with < 200ms latency.
